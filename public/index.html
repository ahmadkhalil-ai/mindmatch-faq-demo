1. User query

Embed the question

The user's question is converted to a vector using an embedding model. In production this uses OpenAI text-embedding-3-small or Supabase AI. Takes ~50ms.

↓
2. Vector search

pgvector cosine similarity

Supabase runs SELECT ... ORDER BY embedding <=> $query LIMIT 3 against your FAQ table. Returns the top 3 matches with similarity scores.

↓
3. Context injection

Build the LLM prompt

Top 3 FAQ entries are injected into a system prompt. If similarity is below 0.70, the bot says it doesn't know rather than hallucinating.

↓
4. Claude API

Generate natural response

Claude Haiku receives the context and user question. Typical total latency: 1.2–2.5 seconds end to end.

↓
5. Logging

Save to database

Query, matched FAQ, similarity score, and response are saved. Low-score queries are flagged so admins know what's missing from the knowledge base.
